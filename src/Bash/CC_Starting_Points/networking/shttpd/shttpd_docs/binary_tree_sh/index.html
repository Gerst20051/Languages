binary_tree.sh

Overview

A binary tree algorithm written in a shell script. The main functions of interest are newTree, deleteTree, insertKey, insertKeyNumeric, treeSearch, treeSearchNumeric, iterateTree, and mergeTrees.

This is a minimal binary tree implementation that does not support removing existing values from the tree once inserted. However, such functionality can be trivially retrofitted on top by adding or clearing a "deleted" attribute on nodes using setTreeField if desired.

To use this shell script, source it after setting DISABLE_TESTS to "true". To run tests, execute the script directly.

insertKey()
Parameters
TREE

The name of the binary tree.

KEY

The key to insert.

Return Value

Obtain the node name of the newly created node using getLastNodeName.

Discussion

If a node already exists with this value, the existing node is returned.

This tree insertion uses string comparisons. You must use treeSearch with this function (and not treeSearchNumeric. For numeric searches, use insertKeyNumeric.

iterateTree

Iterates through a subtree, calling a function for each node. 

iterateTree()
Parameters
TREE

The tree to traverse.

ACTION

The function to call on each node.

CALLONROOT

Set to 1 if you want to also call ACTION on the (bogus) root node. This is usually only set for debug printing purposes.

Discussion

For each node in the tree (in sorted order), the function specified by ACTION is called with a single parameter containing the node name of the node being traversed.

mergeTrees

Copies all of the keys in one tree into another. 

mergeTrees()
Parameters
TREE_SRC

The source tree to copy.

TREE_DST

The destination tree into which the source tree is copied.

Discussion

For each key in TREE_SRC, an equivalent key is inserted in TREE_DST, including any field values associated with it. In the event of a collision for a given key, the resulting set of field values for that key is the union of the two sets of field values, with the new values from TREE_SRC taking precedence.
